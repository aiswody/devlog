## 무엇을 하려 했나

ICU 입실 후 48시간 데이터로 원내 사망률을 예측하는 모델을 만들고 있다.
모델보다 먼저 부딪힌 문제는 데이터였다. MIMIC-IV는 테이블이 30개가 넘고,
같은 "심박수"도 `chartevents`에 여러 `itemid`로 흩어져 있다.
이 글은 원본 CSV에서 모델 입력 행렬을 만들기까지의 전처리 과정을 정리한 기록이다.

## 무엇을 했나

전처리를 세 단계로 나눴다.

1. **코호트 정의** — 성인(18세 이상), 첫 ICU 입실, 입실 후 48시간 이상 체류한 환자만 남긴다.
2. **피처 추출** — 활력징후·검사 수치를 `itemid` 매핑 테이블로 통합하고, 1시간 단위로 리샘플링한다.
3. **결측 처리** — 시계열은 forward fill 후 코호트 중앙값으로 채우고, 결측 마스크를 별도 피처로 남긴다.

| 단계 | 입력 | 출력 |
| --- | --- | --- |
| 코호트 정의 | icustays, patients | 대상 stay_id 목록 |
| 피처 추출 | chartevents, labevents | stay × 시간 × 피처 텐서 |
| 결측 처리 | 피처 텐서 | 학습용 행렬 + 마스크 |

## 어떤 기술을 어떻게 썼나

`chartevents`는 수억 행이라 통째로 읽을 수 없다. pandas의 `read_csv(chunksize=...)`로
청크 단위로 읽으면서 대상 코호트의 행만 걸러 parquet으로 저장했다.

```python title="filter_chartevents.py"
import pandas as pd

USECOLS = ["stay_id", "itemid", "charttime", "valuenum"]

def filter_chartevents(path: str, stay_ids: set[int]) -> pd.DataFrame:
    chunks = []
    for chunk in pd.read_csv(path, usecols=USECOLS, chunksize=5_000_000):
        chunks.append(chunk[chunk["stay_id"].isin(stay_ids)])
    return pd.concat(chunks, ignore_index=True)
```

리샘플링은 `groupby(["stay_id", pd.Grouper(key="charttime", freq="1h")])`로 처리했고,
집계 함수는 피처별로 다르게(혈압은 평균, GCS는 최솟값) 적용했다.

## 배운 점

- 전처리 각 단계의 출력을 parquet으로 저장해 두니, 뒷단계만 다시 돌릴 수 있어 반복 실험이 훨씬 빨라졌다.
- 결측은 "채우는 것"보다 "결측이었다는 사실을 남기는 것"이 중요했다. 마스크 피처를 추가하자 AUROC가 소폭이지만 일관되게 올랐다.
- 코호트 정의를 코드가 아니라 문서로만 남기면 나중에 재현이 안 된다. 정의 자체를 함수로 만들어 버전 관리에 넣었다.
