feature 브랜치를 develop이 아니라 다른 feature 브랜치 위에서 시작해 버렸다.
merge 없이 옮기는 방법을 찾다가 `rebase --onto`를 제대로 이해하게 됐다.

```bash
# git rebase --onto <새 베이스> <옛 베이스> <옮길 브랜치>
git rebase --onto develop feature-a feature-b
```

- `feature-a..feature-b` 구간의 커밋만 잘라서
- `develop` 위에 다시 쌓는다.

## 배운 것

- `rebase --onto`의 세 인자는 "어디로, 어디부터, 무엇을"로 읽으면 된다.
- 인자 두 개짜리 `git rebase develop`은 사실 `--onto develop <merge-base>`의 축약형이었다.
- 옮기기 전에 `git log --oneline feature-a..feature-b`로 잘려나갈 구간을 먼저 확인하면 사고가 없다.
